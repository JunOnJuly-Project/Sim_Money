## KEEP (잘된 것)

- **TDD 슬라이스 커밋**: 포트 정의 → 실패 테스트 → 구현 순서를 W1부터 일관되게 유지. 회귀가 없었다.
- **헥사고날 경계**: `market_data`, `similarity` 모두 포트/어댑터 경계가 명확해 Fake 교체가 쉬웠다.
- **에이전트 역할 분리**: planner → tester → developer → reviewer 파이프라인이 L3 스켈레톤 오염을 막는 게이트 역할을 했다.
- **HANDOFF 동기화**: 각 주차 머지마다 HANDOFF.md 갱신 + 원격 푸시로 세션 단절 없이 이어받기 가능했다.
- **import-linter CI**: 계약 위반을 로컬/CI 양쪽에서 잡아 경계 침범 0건.

## PROBLEM (아쉬운 것)

- **env-check 훅 버그**: 초반 `pre-commit` 훅이 `.env` 부재 시 오탈자로 exit 0 대신 exit 1 발생. `--no-verify` 남발로 훅 신뢰도 하락. → W2 중반에야 수정.
- **FakeStrategy score_map 키 충돌**: `FakeStrategy` 픽스처에서 동일 키를 두 테스트가 재사용해 flaky 발생. 픽스처 격리 원칙 미적용.
- **ADR-002 vs 구현 불일치**: ADR-002 에 명시된 `score_map` 반환 타입이 구현체와 달라 W3 통합 테스트에서 발견. M1 중에 반영했으나 초기 설계 검토가 부족했다.
